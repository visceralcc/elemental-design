# Elemental Design — Web POC · Technical Specification

**Architecture, data model, token system, and rendering for the web POC**

Version 0.1 | April 2026

---

## 1. Architecture

The POC is a single HTML file with embedded CSS and JavaScript. No framework, no build step, no dependencies beyond Google Fonts.

### Why single-file
The demo needs to be shareable with zero friction — email it, airdrop it, drop it on a server, open it in a browser. A single `.html` file achieves this. The tradeoff is that the file will grow as features are added, but for a POC this is the right call. If the file exceeds ~2000 lines, consider extracting CSS and JS into separate files in the same directory.

### Runtime flow
1. Page loads → CSS parsed → Google Fonts fetched → JS runs `render()`
2. `render()` calls three functions: `renderElements()`, `renderCanvas()`, `renderControls()`
3. User interactions (tap, toggle, scale) mutate the `COMPONENTS` array and `state` object, then call `render()` again
4. All rendering is innerHTML-based — no virtual DOM, no diffing. For this scale of UI, full re-render is fast enough.

## 2. Data Model

The POC mirrors the Elemental Design object model from `Spec_ObjectModel.md`. Each component is a plain JavaScript object.

### Component structure

```javascript
{
  id: string,            // unique identifier
  name: string,          // display name in Elements panel
  subtype: string,       // 'shape' | 'information' | 'widget'
  placed: boolean,       // whether it's on the canvas
  isIconRow: boolean,    // (optional) special rendering for icon components
  params: {
    size: { w: number|string, h: number|string },  // 'hug', 'fill', 'auto', or pt
    padding: { top, right, bottom, left },          // all in pt
    color: string,                                   // hex, rgba, or 'transparent'
    cornerRadius: number,                            // pt
    clip: boolean,
    spacing: number,                                 // pt between children (containers)
    scalesWithContent: boolean,
    direction: string,                               // 'vertical' | 'horizontal'
    text: { content, styleRole } | undefined,        // Information subtypes
    image: { src, fit } | undefined,                 // Information subtypes
  },
  children: array        // nested components (containers only)
}
```

### Subtype defaults

These must match `Spec_ObjectModel.md` §8:

| Parameter | Shape | Information | Widget |
|-----------|-------|-------------|--------|
| Size | 200×200pt | Hug content | Hug content |
| Padding | 16pt all | 0 all | 12h / 8v |
| Color | System background | Transparent | System accent |
| Corner Radius | 12pt | 0 | 8pt |
| Clip | On | Off | On |
| Scales with Content | On | On | On |

The POC's Card Container overrides some Shape defaults (340w, 20pt padding, 16pt corner radius, custom fill) to create a realistic card. These overrides represent what a user or Agent would set — the defaults are still the foundation.

### Text style roles

From `Spec_ObjectModel.md` §7.1:

| Role | Size | Weight | Line Height | Letter Spacing |
|------|------|--------|-------------|----------------|
| Display | 36pt | 700 | 1.15 | -0.5px |
| Title | 24pt | 600 | 1.25 | -0.3px |
| Body | 16pt | 400 | 1.5 | 0 |
| Label | 13pt | 500 | 1.3 | 0.2px |
| Mono | 14pt | 400 | 1.5 | 0 |

## 4. Rendering

### Elements panel rendering
- `renderElements()` builds the entire panel as an HTML string and sets `innerHTML`
- Each component row is generated by `renderRow(comp, depth)`
- Depth controls left margin indentation (14pt per level)
- Row background gradient determined by state: unplaced (gray), placed (cyan), selected (blue)
- Expanding a row appends the output of `renderParams(comp)` — a monospace parameter readout grouped by Structure / Content

### Canvas rendering
- `renderCanvas()` builds the canvas view and sets `innerHTML`
- The container is rendered as a flex column with children
- Each child is rendered by `renderCanvasChild(comp, scale, showPad, showSpace)`
- Scale factor is applied as a multiplier on all pt values (size, padding, spacing, font size, corner radius, icon size)
- Padding overlays are absolutely positioned divs with dashed borders and pt labels
- Spacing indicators are placed between children as flex items with a centered pink line and label

### Scale system
Three sizes: S (0.75×), M (1×), L (1.25×). Scale multiplies all rendered pt values. This demonstrates `scalesWithContent` behavior — the container grows/shrinks proportionally with its children.

## 5. State Management

Global state is minimal:

```javascript
state = {
  selected: string|null,    // id of currently selected component
  scale: number,             // 0.75, 1.0, or 1.25
  expanded: { [id]: bool }   // which rows are expanded in Elements panel
}
```

Component placement state lives on each component object (`placed: boolean`). This is mutated directly and `render()` is called afterward. No immutability constraints — this is a simple demo.

## 6. Interaction Model

| User action | What happens |
|-------------|-------------|
| Tap unplaced row | Component's `placed` becomes true; it appears on canvas and nests under container in panel |
| Tap placed row | Component becomes selected; row highlights blue; canvas shows selection outline |
| Tap placed row again | Parameter view expands/collapses (>>> / ^^^) |
| Tap canvas component | That component becomes selected |
| Tap empty canvas | Selection cleared |
| Toggle Padding checkbox | Padding overlays shown/hidden on selected component |
| Toggle Spacing checkbox | Spacing indicators shown/hidden between children |
| Tap scale button (S/M/L) | All rendered sizes multiply by scale factor |
| Tap "Place All" | All components placed at once |

## 7. Mobile Behavior

At viewport widths ≤640px:
- Main layout switches from `flex-direction: row` to `column`
- Elements panel fills full width, max-height 45vh
- Canvas fills remaining space below
- Mode bar pills shrink slightly
- Touch targets remain ≥44pt for accessibility
